# Page Archetype; Login Signup Page
WORKFLOW ==> page-archetype-login-signup-page

## Purpose
A compact authentication page optimized for clarity, trust, and low friction.

## Typical section order
1. Brand or logo
2. Welcome headline
3. Short helper text
4. Form
5. Primary submit button
6. Secondary auth option if relevant
7. Terms or trust note
8. Switch login signup link

## Hero structure
Usually no large hero is needed. If a split-screen layout is used, keep the visual panel simple and supportive.

## Content priorities
- Minimize friction
- Make the form readable and obvious
- Reduce distractions
- Support mobile first
- Reinforce trust and security

## Common components
- Email field
- Password field
- Show password toggle
- Remember me and forgot password row
- Submit button
- Divider and SSO option

## Layout notes
- Keep form width compact
- Use 1 clear visual path
- Ensure strong input focus states
- Prefer a centered card or clean split screen

## Do
- use clear labels
- use visible error states
- keep the page lightweight
- support keyboard navigation

## Avoid
- excessive decoration
- overloaded side content
- ambiguous labels
- hidden primary CTA
