# Page Archetype; SaaS Landing Page
WORKFLOW ==> page-archetype-saas-landing-page

## Purpose
A high-conversion marketing page for a software product.

## Typical section order
1. Hero
2. Social proof or logo strip
3. Problem and solution
4. Feature grid
5. Product preview
6. Benefits or use cases
7. Testimonials
8. Pricing teaser or pricing section
9. FAQ
10. Final CTA footer

## Hero structure
- Eyebrow
- Headline
- Supporting paragraph
- Primary CTA
- Secondary CTA
- Product image, mockup, or bento panel

## Content priorities
- Make the product promise immediately clear
- Show value quickly
- Use concise copy
- Add proof early
- Keep one primary CTA path

## Common components
- Hero CTA cluster
- Feature cards
- Stats strip
- Product screenshot section
- Testimonial cards
- FAQ accordion

## Layout notes
- Use a clear hero split layout
- Keep max width around 1180px to 1240px
- Use visual rhythm between proof, explanation, and CTA sections
- Avoid overly long paragraphs

## Do
- front-load value proposition
- use proof near the top
- make CTAs consistent
- group features into 3 to 6 cards

## Avoid
- generic hero copy
- giant unbroken feature lists
- too many CTA variants
- weak product visuals
