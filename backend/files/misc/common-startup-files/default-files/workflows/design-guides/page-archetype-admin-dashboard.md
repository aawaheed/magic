# Page Archetype; Admin Dashboard
WORKFLOW ==> page-archetype-admin-dashboard

## Purpose
A high-clarity application shell for managing data, workflows, and insight-heavy tasks.

## Typical section order
1. App shell with sidebar or top nav
2. Page header with title and actions
3. KPI or status cards
4. Main work area
5. Secondary panels or activity feed
6. Tables, charts, or forms
7. Utility footer if needed

## Content priorities
- Speed of scanning
- hierarchy of actions
- clear grouping of data
- consistency of components
- reduced cognitive load

## Common components
- Sidebar navigation
- Header actions
- Metric cards
- Data tables
- Filters
- Search input
- Modals and drawers
- Empty states

## Layout notes
- Use a stable shell
- Keep strong spacing rhythm
- Keep action buttons predictable
- Prioritize readability over decoration
- Use elevation sparingly

## Do
- keep navigation obvious
- make states clear
- use muted surfaces for grouping
- support dense but readable layouts

## Avoid
- unnecessary motion
- visual clutter
- too many competing accents
- oversized headers that waste space
